How I ship a tiny RAG for an SMB (without drama)

The smallest thing that works, then earn the next step.

1) Map the data

Where is it, who owns it, what’s private/PII, and how often does it change? You can’t skip this.

2) Retrieval before prompts

Chunking + embeddings + metadata filters. Pick a vector store you can actually run (pgvector is great).

3) One guarded workflow

Start with a single task the team cares about. Add an approval step and log everything.

4) Honest evaluation

Keep a tiny eval set. Track answer quality and cost. If it’s not better than “Ctrl+F,” fix retrieval.

5) Ship, then simplify

Remove knobs nobody uses. Automate ingestion. Only add features when the last one is clearly working.

Tools I like: FastAPI, Streamlit, Postgres/pgvector, Docker, Ollama/OpenAI.
